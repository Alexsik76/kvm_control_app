# Development Progress: IP-KVM Client

## 1. Поточний стан (Phase 1: Video Pipeline - ЗАВЕРШЕНО)

Реалізовано стабільну, модульну архітектуру відео-конвеєра. Клієнт здатний підключатися до RTSP (TCP) або UDP потоків, декодувати H.264 у фоновому режимі та відображати відео з низькою затримкою без вильотів.

### Архітектура (SOLID / SRP)
*   **FFmpegStreamNode**: Повністю ізольований вузол для роботи з FFmpeg (libavformat/libavcodec). Займається мережевим з'єднанням, демультиплексуванням та фоновим декодуванням. Не залежить від графічного інтерфейсу.
*   **SDLVideoDecoder**: Модуль рендерингу. Отримує розпаковані YUV-кадри від стрим-вузла та завантажує їх у VRAM через `SDL_UpdateYUVTexture`.
*   **OverlayGUI**: Відокремлений компонент для керування оверлеєм Dear ImGui. Забезпечує відображення статусу стрімінгу та елементів керування.
*   **Main Coordinator**: Мінімізована точка входу, яка лише керує життєвим циклом компонентів та головним циклом подій.

### Технологічний стек
*   **Система збірки**: CMake + MSVC.
*   **Залежності**: SDL2 (Відео/Події), FFmpeg 8.1 (Декодування), Dear ImGui (GUI).
*   **Протоколи**: Підтримка RTSP (через TCP) та сирого UDP.
*   **Декодування**: Оптимізоване програмне (CPU) декодування YUV420P. Обрано для 100% стабільності та уникнення конфліктів потоків у драйверах D3D11 при фоновій роботі.

## 2. Як запускати та тестувати

### Запуск з аргументами
Клієнт тепер приймає URL потоку як аргумент командного рядка:
```powershell
# Запуск для RTSP (через MediaMTX)
.\build\KVMControlApp.exe "rtsp://admin:password@localhost:8554/kvm"

# Запуск для UDP (стандартний режим)
.\build\KVMControlApp.exe "udp://127.0.0.1:5000"
```

### Тестовий стрім (RTSP)
Якщо у вас запущений RTSP сервер (напр. Mediamtx):
```bash
ffmpeg -re -f lavfi -i testsrc=size=1280x720:rate=60 -c:v libx264 -preset ultrafast -tune zerolatency -f rtsp rtsp://localhost:8554/kvm
```
