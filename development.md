# Development Progress: IP-KVM Client

## 1. Поточний стан (Phase 1: Video Pipeline - ЗАВЕРШЕНО & ОПТИМІЗОВАНО)

Реалізовано стабільну, модульну архітектуру відео-конвеєра. Клієнт переведено на **SDL3**, що забезпечує сучасну базу для роботи з вікнами, подіями та рендерингом.

### Архітектура (SOLID / SRP)
*   **FFmpegStreamNode**: Повністю ізольований вузол для роботи з FFmpeg. 
    *   **Швидкий старт**: Додано передачу параметрів кодека (`avcodec_parameters_to_context`), що дозволяє миттєво починати декодування UDP/RTSP потоків без очікування in-band пакетів.
    *   **Thread Safety**: Реалізовано безпечний `interrupt_callback` з атомарним прапорцем для запобігання Race Condition при закритті програми під час блокуючого читання мережі.
*   **SDLVideoDecoder**: Модуль рендерингу на базі **SDL3**. Отримує YUV-кадри та завантажує їх у VRAM. Дотримано логічної константності (GetTexture більше не використовує `const_cast`).
*   **OverlayGUI**: Компонент оверлею на базі **Dear ImGui v1.91.5**. Адаптовано під SDL3 та SDL_Renderer3.
*   **Main Coordinator**: Точка входу, адаптована під SDL3 (оновлено ініціалізацію та цикл подій).

### Технологічний стек
*   **Система збірки**: CMake + MSVC (Visual Studio 2026 / v180).
*   **Залежності**: 
    *   **SDL3** (Відео/Події) — основний фреймворк.
    *   **FFmpeg 8.1** (Декодування).
    *   **Dear ImGui v1.91.5** (GUI) — версія з повною підтримкою SDL3.
*   **Протоколи**: RTSP (TCP), сирий UDP.
*   **Декодування**: Оптимізоване програмне (CPU) декодування H.264 (YUV420P/YUVJ420P).

## 2. Як запускати та тестувати

### Збірка
Використовуйте оновлений скрипт збірки, який автоматично налаштовує оточення MSVC та пресети CMake:
```powershell
.\build.bat
```

### Локальне тестування (FFmpeg Test Source)
Для зручного тестування без зовнішнього джерела використовуйте готовий скрипт, який запускає генерацію тестового відео (UDP, 720p@60fps):
```powershell
.\start_stream.bat
```
Після цього запустіть клієнт:
```powershell
.\build\Debug\KVMControlApp.exe "udp://127.0.0.1:5000"
```
