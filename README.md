# KVM Video Codec (Native Library)

Це високопродуктивна C++ бібліотека для отримання, депакетизації та декодування відеопотоку через WebRTC. Вона спроектована як проміжний шар (bridge) між нативним WebRTC/FFmpeg стеком та високорівневими додатками (наприклад, на базі .NET/Avalonia).

## 📂 Структура проекту

Проект розділений на логічні модулі за принципом SRP (Single Responsibility Principle):

*   **`include/KVMVideoCodec.h`** — Публічний C-сумісний API. Використовується для ініціалізації (`KvmInitialize`), зупинки (`KvmStop`) та отримання кадрів.
*   **`network/`** — Модуль для роботи з сигналізацією.
    *   `IHttpClient`: Абстракція HTTP-клієнта.
    *   `HttpClient`: Реалізація на базі WinHTTP (Windows) для обміну SDP та ICE-кандидатами з сервером FastAPI.
*   **`video/`** — Модуль обробки відеопотоку.
    *   `WebRTCStreamNode`: Основний вузол, що керує `libdatachannel` (WebRTC) та циклом декодування.
    *   `RtpDepacketizer`: Відповідає за збірку фрагментованих RTP-пакетів (RFC 6184) у чисті H.264 NAL юніти.
    *   `IVideoDecoder`: Інтерфейс для майбутніх апаратних декодерів.

## 📥 Вхідні дані (Input)

Бібліотека взаємодіє з сервером сигналізації та стрімінгу у два етапи:

1.  **Signaling (HTTP JSON):**
    *   Надсилає: `SDP Offer` та `ICE Candidates`.
    *   Отримує: `SDP Answer` та `ICE Candidates` від віддаленої сторони.
2.  **Media Stream (WebRTC/RTP):**
    *   Протокол: **WebRTC** (через UDP/DTLS).
    *   Кодек: **H.264**.
    *   Контейнер: **RTP** (RFC 3550).
    *   Схема пакування: **RFC 6184** (підтримка Single NAL, STAP-A та FU-A фрагментації).

## 📤 Вихідні дані (Output)

Результатом роботи бібліотеки є розпаковані та декодовані відеокадри, які передаються у хост-додаток через callback-функцію:

*   **Формат пікселів:** `BGRA8888` (32 біти на піксель).
*   **Колірна модель:** Здійснюється автоматична конвертація з `YUV420p` (FFmpeg) у `BGRA` за допомогою `libswscale`.
*   **Передача даних:** Прямий доступ до буфера пам'яті (`uint8_t*`).
*   **Параметри кадру:** Передаються `width`, `height` та `stride` (крок рядка в байтах), що дозволяє уникнути проблем з вирівнюванням пам'яті при рендерингу.

## 🛠 Збірка

Для збірки використовується CMake з пресетами:
1. Запустіть `build.bat` для автоматичної конфігурації та компіляції.
2. Бібліотека `KVMVideoCodec.dll` та її залежності (FFmpeg, LibDataChannel) будуть скопійовані у вихідну папку проекту.

**Залежності:** `ffmpeg`, `libdatachannel`, `nlohmann-json` (керуються через `vcpkg`).
