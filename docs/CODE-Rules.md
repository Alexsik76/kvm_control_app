# Вимоги до якості коду

## Правила генерації C++ коду

1. Процес взаємодії

Не генеруй код без мого прямого прохання.

Не генеруй реалізацію всього коду одразу (працюємо ітеративно).

При внесенні змін, які не перевищують 5 рядків, пропонуй лише змінений фрагмент із прив'язкою до місця, а не весь файл цілком.
2. Архітектура та стандарти

Застосовуй сучасні галузеві стандарти (C++17/20).

Дотримуйся принципів SOLID, особливо SRP (Single Responsibility Principle): кожен клас чи функція повинні мати лише одну зону відповідальності. Уникай "божественних об'єктів".

Функції мають бути короткими і виконувати суворо одну задачу без прихованих побічних ефектів.

Використовуй RAII: жодного ручного керування пам'яттю (new/delete). Завжди використовуй розумні вказівники (std::unique_ptr, std::shared_ptr) та контейнери STL.
3. Чистота коду та коментарі

Усі коментарі в коді повинні бути написані виключно англійською мовою.

Код як документація: використовуй змістовні назви для змінних та функцій, щоб логіка читалася як текст.

Без зайвого шуму: не пиши коментарі, які пояснюють що робить код. Коментуй лише неочевидні рішення або хаки (пояснюй чому це зроблено).

Ніколи не залишай закоментовані шматки старого коду.
