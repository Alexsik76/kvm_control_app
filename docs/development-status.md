# Development Progress: IP-KVM Client

## 1. Поточний стан (Phase 1 & 2: Video Pipeline & HID Interaction - ЗАВЕРШЕНО)

Реалізовано стабільну, модульну архітектуру відео-конвеєра та підсистеми введення. Клієнт переведено на **SDL3**, що забезпечує сучасну базу для роботи з вікнами, подіями та рендерингом, а також імплементовано мережевий HID-клієнт.

### Основні досягнення (Phase 2):
*   **Mouse Capturer**: Реалізація `SDL_SetRelativeMouseMode` (SDL3) для захоплення миші.
*   **Event Mapping**: Трансляція скан-кодів SDL3 у HID-репорти Raspberry Pi.
*   **WebSocket Client**: Інтеграція клієнта для відправки команд керування з фоновою чергою для уникнення затримок.

### Архітектура (SOLID / SRP)
*   **KVMApplication**: Головний оркестратор, що пов'язує ізольовані модулі відео, вводу та інтерфейсу без сильної зв'язності (Loose Coupling).
*   **FFmpegStreamNode**: Повністю ізольований вузол для роботи з FFmpeg (швидкий старт, thread-safe переривання).
*   **SDLVideoDecoder**: Модуль рендерингу на базі **SDL3**, що завантажує YUV-кадри у VRAM.
*   **SDLInputCapturer**: Модуль перехоплення вводу через SDL3 (Relative Mouse Mode, Scancodes).
*   **EventMapper**: Відповідає виключно за трансляцію SDL Scancodes та модифікаторів у стандартизовані коди USB HID.
*   **WebSocketHIDClient**: Асинхронний мережевий клієнт на базі **IXWebSocket**. Використовує фоновий потік (Worker Thread) та потокобезпечну чергу повідомлень для швидкої серіалізації (через `snprintf`) без блокування основного циклу рендерингу.
*   **OverlayGUI**: Компонент оверлею на базі **Dear ImGui v1.91.5**. Відображає статус мережі та HID-телеметрію (абсолютні координати миші та останній натиснутий сканкод).

### Технологічний стек
*   **Система збірки**: CMake + MSVC (Visual Studio 2026 / v180).
*   **Залежності**: 
    *   **SDL3** (Відео/Події) — основний фреймворк.
    *   **FFmpeg 8.1** (Декодування).
    *   **Dear ImGui v1.91.5** (GUI).
    *   **IXWebSocket** (Мережа / WebSocket).
*   **Протоколи**: RTSP (TCP), сирий UDP (Відео), WebSocket (HID).

## 2. Як запускати та тестувати

### Збірка
Використовуйте скрипт збірки, який автоматично налаштовує оточення MSVC та пресети CMake:
```powershell
.\build.bat
```

### Запуск з аргументами
Клієнт приймає URL відеопотоку та URL WebSocket сервера як аргументи:
```powershell
.\build\Debug\KVMControlApp.exe "udp://127.0.0.1:5000" "ws://127.0.0.1:8080"
```
